# DB 직접조회 vs 레디스 캐시 조회 성능비교
- Look Aside + Write Around 캐시전략 사용,
- 쓰기는 DB로 바로 전달
- 읽기는 레디스 캐시 조회를 해보고 데이터 없으면 DB에서 조회 -> 레디스 저장 -> 데이터 응답

- 비교 결과 단건 조회에서, 레디스가 2배 에서 3배이상 조회 소요시간이 짧음.
- 전체 조회에서, 데이터수가 늘어날수록 첫 조회는 레디스에 없는 데이터를 저장하느라 느림. 그러나 데이터 저장후, 레디스가 DB 직접 조회보다 3배에서 10배 까지 빨랐고, 데이터가 늘어날수록 그 차이는 커짐.
